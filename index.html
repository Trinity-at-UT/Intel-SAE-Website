Intel Simulation and Analysis Engine (SAE)

Changes the way you look at the CPU.

Find Out More

Dynamic Binary Instrumentation of OS Kernel, Driver and BIOS

We introduce the Intel速 Simulation and Analysis Engine (Intel速 SAE) --- a framework for full-system instruction-level instrumentation of "ring 0" (privileged) and "ring 3" (user-level) code behavior on x86 platforms. When plugged-in to a Wind River速 Simics Virtual Platform, Intel速 SAE boots native operating systems (e.g. Linux and Windows, as well as Android), running unmodified binaries while facilitating flexible and customizable instruction-level instrumentation of everything executing on the CPU, i.e. BIOS, kernel, drivers and all kernel and user-space processes. Beyond its ability to instrument a single system, Intel SAE is capable of distributed node-to-node multi-system simulation and instrumentation, useful for analysis of enterprise-scale workloads, such as CloudSuite, Hadoop, Memcached etc.

Chapter 2: Instruction instrumentation

Learn how to instrument instruction execution.

We introduce Intel SAE's capabilities, and help you get started with setting up and configuring your environment, and show you the most basic know-hows that will help you get off to a quick start. Build and run your first ztool!

Chapter 3: Memory Instrumentation

Track memory read and write behavior.

Intel SAE provides powerful APIs that allow you to monitor all CPU memory accesses. Discover the APIs that allow you to monitor read/write instructions, trace the virtual, linear and physical address of each memory instruction, etc.

Chapter 4: Interrupt and Exception instrumentation

Track memory read and write behavior.

Intel SAE provides powerful APIs that allow you to monitor all CPU memory accesses. Discover the APIs that allow you to monitor read/write instructions, trace the virtual, linear and physical address of each memory instruction, etc.

Chapter 5: OS instrumentation

Observe and modify simulated guest OS behavior.

Intel SAE comes with APIs that make it easy to observe the behavior of the simulated guest operating system (and its processes).

You can use the available high-level APIs to track guest OS functions that are executed, track processes and threads as they are created and destroyed etc. Get detailed information, such as process and thread IDs or the symbolic name of the process.

Only supports the Linux operating system. Extensible to other operating systems through user-defined ztool APIs

Chapter 7: Advanced Concepts

There are numerous advanced features within Intel SAE that are available for advanced developers, some of these address correctness issues when building complex (multi)instrumentation tools and others belong to advanced instrumentation capabilities.

We cover features like "dynamic reconfiguration" and "event callback order".
